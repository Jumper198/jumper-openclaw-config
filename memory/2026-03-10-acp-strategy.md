# ACP (Agent Coding Protocol) Strategy & Application (2026-03-10)

## 核心认知升级 (ACP 降维打击)
- **多 Bot 方案的谬误**：常驻多 Bot 互相交互会引发严重的上下文污染与 Token 失控，属于低效的“马戏团”模式。
- **OPC 极简架构的终极解**：采用“主线程（赵无敌）+ 按需派生隔离子线程（Subagent/ACP）”的动态算力分配模型。任务用完即毁，无状态派生。

## ACP (独立编码智能体引擎) 运行机制
1. **黑灯工厂模式**：通过 `sessions_spawn(runtime="acp", pty=true)` 唤醒专家 Agent（如 Claude Code/Codex），并赋予 PTY 终端读写权限。
2. **完全自治闭环**：
   - 自动创建文件、安装依赖、编写代码。
   - 自动执行测试脚本，捕获报错输出（Error Log）。
   - 自动分析报错原因，自我修正代码并重试，直至流程跑通。
3. **主线程隔离**：主线程（赵无敌）在此期间不干预，待子线程完成全部闭环后，再将最终结果汇总汇报，极大地节省了主线程的通信成本和用户心智。

## 算力套利终极逻辑 (针对上海徐汇区补贴)
- ACP 在高频试错与自我修正的闭环中，将产生巨大的 Token 与本地算力消耗。
- 核心商业模式（数字劳动套利）：接取高毛利的技术外包订单，利用政府发放的 100万模型券+100万算力券覆盖 ACP 运行成本。政府买单 OpEx，个人获取全额利润现金。
