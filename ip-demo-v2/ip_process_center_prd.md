流程中心（对标 Innojet）产品 & 技术方案 v0.1 & v0.2
面向：企业知识产权/法务/研发管理团队（非代理所视角）

1. 核心对象（数据模型层面）
- 流程模板（WorkflowTemplate）：业务类型、节点定义、角色、SLA/期限规则、表单结构。
- 流程实例（WorkflowInstance）：由模板发起的一次具体业务（例如“专利申请-某项目”）。
- 任务（Task）：实例在某节点产生的待办；支持主办/协办/抄送。
- 节点（Node）：实例所处阶段；支持审批/办理/并行/会签。

2. 核心页面一：待办/已办列表页 (TaskListPage)
- 顶栏卡片 (Stats Cards)：突出显示「逾期任务（红）」、「近期到期（黄）」、「总待办（蓝）」。
- 高级检索区 (FilterPanel)：快捷过滤组 (全部 / 逾期 / 今日 / 5天内)。折叠表单查询（主题、流程名称、发起人、时间段）。
- 数据表格组件 (WorkflowTable)：必显列：流程分类、主题、当前节点、当前处理人、办理期限（结合颜色预警）。行操作列：办理 / 查看。

3. 核心页面二：新建提案大表单页 (CreateProposalForm)
- 基本信息区块：提案联系人（必填）、申请类型（必填，默认发明）、二级单位（必填）、部门领导（必填）、专利工程师（必填）、提案编号、提案日（选填）。
- 提案信息区块：提案名称（必填）、专利提案核心内容、关联项目。
- 费用承担区块：支持多部门分摊：配置 承担部门 及 承担比例(%)。
- 发明人信息区块：第一发明人、共同发明人列表（支持添加多人，包含：发明人工号、发明人、国籍、部门、是否公开、奖励比例(必填，总和须100%)）。
- 底部悬浮操作区：取消、暂存草稿、启动流程(高亮)。

4. 技术要求
前端采用 React/Next.js + TailwindCSS + Ant Design 或其他 UI 库。
后端采用 Node.js (Next.js API routes) + SQLite (Prisma 或 better-sqlite3) 作为 MVP 演示数据源。
